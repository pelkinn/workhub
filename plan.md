# WorkHub — 4-месячный план развития

Формат плана:
- один проект,
- одна задача в день (≤ 2 часа),
- каждая задача имеет артефакт,
- каждая неделя имеет измеримый результат.

---

## МЕСЯЦ 1 — Выход в прод

### Неделя 1 — Контракты и скелет
**Результат недели:** backend и frontend запускаются локально, есть auth.

- **Пн** — README с MVP и границами проекта  
  Артефакт: `README.md`
- **Вт** — доменная модель  
  Артефакт: `docs/schema.md`
- **Ср** — инициализация backend  
  Артефакт: NestJS + Postgres + миграции
- **Чт** — auth endpoints  
  Артефакт: `/auth/login`, `/auth/me` (refresh опционально)
- **Пт** — минимальный frontend  
  Артефакт: Nuxt + login + `/me`

---

### Неделя 2 — RBAC и приглашения
**Результат недели:** проект можно безопасно шарить.

- **Пн** — RBAC матрица  
  Артефакт: `docs/rbac.md`
- **Вт** — membership guard  
  Артефакт: `membership.guard.ts`
- **Ср** — создание приглашения  
  Артефакт: `POST /projects/:id/invite`
- **Чт** — принятие приглашения  
  Артефакт: `POST /invites/:token/accept`
- **Пт** — e2e тест RBAC  
  Артефакт: `rbac.e2e-spec.ts` (1 сценарий)

---

### Неделя 3 — Очереди и уведомления
**Результат недели:** система выполняет фоновые задачи.

- **Пн** — подключение BullMQ  
  Артефакт: тестовый job
- **Вт** — reminder job  
  Артефакт: `reminder.processor.ts`
- **Ср** — daily digest cron  
  Артефакт: job с логированием
- **Чт** — Telegram уведомление  
  Артефакт: отправка одного сообщения
- **Пт** — retries и ошибки  
  Артефакт: обработка failed jobs

---

### Неделя 4 — Интеграции и деплой
**Результат недели:** проект доступен по URL.

- **Пн** — Telegram `/add`  
  Артефакт: задача создаётся из Telegram
- **Вт** — webhook inbox  
  Артефакт: `POST /inbox`
- **Ср** — AuditLog  
  Артефакт: запись create/update/delete
- **Чт** — деплой  
  Артефакт: backend доступен по URL
- **Пт** — финальный README  
  Артефакт: выводы + список улучшений
